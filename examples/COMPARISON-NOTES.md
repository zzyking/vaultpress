## Positioning difference

### md-to-pdf
A mature, general-purpose Markdown → PDF CLI.

### This project
An Obsidian-aware Markdown → PDF exporter focused on note quality and practical rendering of Obsidian-style content.

## Where md-to-pdf is stronger right now

- mature CLI surface
- config/frontmatter story
- watch mode
- broader generic Markdown productization
- release/test maturity

## Where this project is stronger already

- Obsidian wikilinks / refs / embeds
- callout-aware rendering
- real-note export tuning
- Chinese technical-note friendliness
- browser-print output tuned around actual Obsidian-style notes

## Suggested public messaging

Do **not** frame this as:
- "another md-to-pdf"
- "the most generic Markdown PDF CLI"

Frame it as:
- "Obsidian-aware Markdown to PDF exporter"
- "Export Obsidian-style notes to high-quality PDFs"
- "Browser-print-based PDF export for Obsidian-flavored notes"
